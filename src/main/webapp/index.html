Test Java Servlet APIs

This project contains mock APIs designed to challenge code detection tools.

Plain Java Servlets

/servlet/standard/*
GET POST PUT DELETE
Standard HTTP methods implementation
/servlet/multiple/*
GET POST PUT DELETE PATCH
All methods handled identically (discouraged)
/servlet/custom/*
GET POST FUNKYTOWN DANCEPARTY SUPERDUPER
Custom/non-standard HTTP methods
/servlet/missing/*
GET
Missing POST, PUT, DELETE implementations (discouraged)
/servlet/unsafe/*
GET POST
Security issues: no validation, SQL injection risk
/servlet/overloaded/*
All methods via service() override (bypasses standard routing)
/servlet/empty/*
Methods implemented but produce no/empty responses

JAX-RS / Jersey

/jaxrs/standard
GET POST PUT DELETE PATCH
Standard JAX-RS resource
/jaxrs/custom
GET POST PUT FUNKYTOWN DANCEPARTY SUPERDUPER
Custom methods via @HttpMethod
/jaxrs/problematic
Missing annotations, no validation
/jaxrs/mixed
Conflicting annotations, ambiguous mappings

Spring MVC

/spring/standard
GET POST PUT DELETE PATCH
Standard Spring MVC controller
/spring/custom
GET FUNKYTOWN DANCEPARTY SUPERDUPER
Custom methods (manual method checking)
/spring/problematic
Missing annotations, no validation
/spring/mixed
Conflicting annotations, ambiguous mappings

Testing

Use curl or any HTTP client to test these endpoints:

# Standard GET
curl http://localhost:8080/test-servlet-apis/servlet/standard/test

# Custom method (requires HTTP client that supports custom methods)
curl -X FUNKYTOWN http://localhost:8080/test-servlet-apis/servlet/custom/test

# JAX-RS
curl http://localhost:8080/test-servlet-apis/jaxrs/standard

# Spring MVC
curl http://localhost:8080/test-servlet-apis/spring/standard

⚠️ Warning: This project contains intentionally problematic code patterns and should NOT be used in production.
